Company Profile · 2026

Privasimu Nexus

Platform Kepatuhan UU PDP Indonesia — End-to-End, AI-Powered, Multi-Tenant SaaS

Privasimu Nexus adalah platform manajemen kepatuhan Undang-Undang Pelindungan Data Pribadi (UU No. 27 Tahun 2022) yang menyatukan seluruh kebutuhan Data Protection Officer, tim Legal, Compliance, dan IT-Security dalam satu workspace terintegrasi. Dibangun cloud-native untuk organisasi yang punya ribuan subjek data, puluhan sistem informasi, dan regulasi yang tidak berhenti berubah.

03 Modul Penilaian Kepatuhan (Assessment Engine)

Fondasi platform. Setiap organisasi UU PDP wajib punya ROPA, dan bila risk tinggi wajib DPIA. Platform kami mengotomasi seluruh alur wizard + scoring + export.

ROPA Core

Records of Processing Activities — kewajiban inti Pasal 31 UU PDP.

  • Wizard 7 langkah sinkron format KOMDIGI
  • Risk Level otomatis dari 8 trigger wizard
  • Multi-DPO / PIC / System per 1 ROPA
  • Master Data Retensi reusable per tenant
  • Auto-trigger DPIA saat risk HIGH
  • Approval Workflow multi-level
  • Export PDF + DOCX canonical/custom
  • AI Auto-Fill 7 section dari 1 deskripsi
  • Version Diff Viewer untuk audit
DPIA Core

Data Protection Impact Assessment — Pasal 34 UU PDP.

  • 21 kategori risiko wajib
  • Risk Matrix 5×5 color-coded
  • Event Templates per kategori
  • Linkage ROPA cascade high-risk
  • AI Auto-Fill DPIA
  • Export PDF + DOCX dengan risk matrix visual
GAP Assessment Core

Audit kepatuhan berbasis indikator UU PDP.

  • 33 pertanyaan UU PDP resmi
  • Custom Questions per tenant
  • Multi-Regulation: UU PDP, GDPR, PDPA, ISO 27701
  • Evidence Upload per-question
  • Radar + Trend chart
  • Benchmark industri
  • AI Remediation Plan prioritas UU PDP
  • Export PDF + DOCX dengan rekomendasi
LIA Core

Legitimate Interest Assessment — Pasal 20(1)(f).

  • 3-step balancing test: Purpose → Necessity → Balancing
  • Template pre-filled untuk fraud detection, direct marketing, HR analytics
  • Export PDF dokumentasi akuntabilitas
TIA Core

Transfer Impact Assessment — Pasal 56.

  • Country risk scoring otomatis
  • Safeguards library: SCC, BCR, adequacy decisions
  • Linkage ROPA yang transfer internasional
Maturity Assessment Executive

Evaluasi tingkat kedewasaan program privasi.

  • 5 level maturity: Initial → Developing → Defined → Managed → Optimized
  • Domain coverage: Governance, Policy, Awareness, Technology, IR
  • Executive report untuk presentasi Board

04 Hak Subjek Data & Consent

DSR — Data Subject Rights Management PASAL 5–8

Alur permintaan hak subjek data: akses, koreksi, penghapusan, portabilitas, penarikan consent, keberatan, info.

  • 72-jam SLA countdown sesuai Pasal 32 dengan auto-alert ke DPO
  • Multi-channel intake: manual entry, web embed form, email intake (IMAP)
  • 7 tipe request: Access, Correction, Deletion, Portability, Withdraw Consent, Objection, Information
  • Identity verification via email OTP
  • Scope picker — pilih Information System yang kena impact
  • SQL Generator (upcoming) — platform generate SQL SELECT/UPDATE/DELETE; admin bank eksekusi sendiri
  • Execution Log per-system dengan upload bukti
  • Completion Certificate auto-generate PDF signed DPO

Consent Management PASAL 22–25

  • Collection Points — tiap touchpoint (web, app, in-store) dengan kode unik
  • Consent Records — track per-subjek lengkap dengan bukti IP/user-agent
  • Preference Center publik — subjek review + withdraw sendiri
  • Consent Editor Embed — iframe untuk website client
  • Consent Version History — audit trail perubahan T&C
  • Withdrawal Propagation — auto-flag ROPA + CRM linked

Cross-Border Transfer PASAL 56

  • Transfer log per-ROPA
  • Safeguard attachment: SCC, BCR, konsen eksplisit
  • Country adequacy status cek otomatis

07 Data Discovery & Inventory

PII Scanner & Schema Registry

Two-Mode Architecture — platform mengakomodasi baik organisasi yang terbuka dengan scan langsung, maupun bank yang no-third-party-DB-access.

  • Live Scan Mode (self-hosted/SaaS trusted) — connect MySQL/PostgreSQL/MongoDB, discover schema, classify PII per column, sample-scan pattern detection
  • Generate-Only Mode (bank) — admin register schema manual atau import DDL; platform zero koneksi DB; pure SQL generator untuk DSR
  • Split/Sharded DB Support — 1 logical system bisa punya N physical shards (30M customer bank di 4 server); schema sekali, shards metadata saja
  • PII auto-detect: NIK, NPWP, Email, Telepon, Alamat, Rekening, IP, Cookie, Health, Biometric, Anak, Agama, Politik, Orientasi Seksual
  • Classification Engine — pattern + context analysis
  • OCR Scanner — upload dokumen physical (KTP scan, form fisik) untuk PII extract
  • Changelog — drift detection antar scan
  • Integrasi ROPA wizard — populasikan otomatis "jenis data dikumpulkan"

Data Posture Score

  • Unified score agregasi: ROPA coverage, DPIA completion, GAP score, DSR SLA, Breach history, Vendor risk
  • Per-area drilldown
  • Trend over time — grafik improvement

  • Multi-provider: OpenRouter, OpenAI, Anthropic, Gemini, Azure, DeepSeek, Groq

AI Auto-Fill AI

Generate wizard ROPA/DPIA otomatis dari deskripsi natural language.

  • Prompt-aware enumeration — output exact match ke opsi wizard
  • Tenant context injection
  • Chained flow: ROPA high-risk auto DPIA
AI Contract Review AI

Analisis kontrak PDF/DOCX terhadap klausul PDP-critical.

  • Upload → parse → report risk rendah/sedang/tinggi
  • Checklist DPA: liability, data residency, audit rights, breach SLA, subprocessor, termination
  • Diff terhadap template standar tenant
AI Policy Review AI

Analisis SOP, Privacy Policy, T&C terhadap UU PDP.

  • Upload policy → AI compliance gap report
  • Rekomendasi revisi klausul per-pasal UU PDP
AI Remediation Plan AI

Otomasi roadmap penutupan gap dari GAP Assessment.

  • Input: jawaban "Belum"/"Sebagian" + konteks organisasi
  • Output: tahapan prioritas + estimasi effort + PIC recommendation + regulatory reference
